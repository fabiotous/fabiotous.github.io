Creating a Simple Web Page Using GitHub Pages

How I Set Up GitHub Pages

To set up GitHub Pages, I followed these steps:

  1. Created a new repository named fabiotous.github.io:

    This naming convention is required for GitHub to recognize it as a user site and publish it at https://fabiotous.github.io/.

  2. Cloned the repository locally: 
    git clone https://github.com/fabiotous/fabiotous.github.io.git
  3. Created an index.html file:

    This file contains the content of the web page.

  4. Pushed the changes to GitHub: 
    git add .
git commit -m "Initial commit"
git push origin main

How I Built the Page

I wrote the HTML content directly in the index.html file. The page includes:

  • An introduction to the project.
  • Instructions on how I set up GitHub Pages.
  • Details on how I built the page.
  • The difficulties encountered and how I solved them.

Difficulties Encountered and How I Solved Them

During the process, I faced a few challenges:

  1. Problem: Site Not Displaying Immediately
    Solution: Realized that GitHub Pages can take a few minutes to publish the site after pushing changes. I waited for a few minutes and then refreshed the page.
  2. Problem: Incorrect Repository Naming
    Solution: Initially, I named the repository incorrectly. To fix this, I renamed it to fabiotous.github.io to match GitHub Pages' requirement for user sites.
  3. Problem: Styles Not Applying
    Solution: Ensured that the CSS styles were correctly placed within the <style> tags inside the <head> section of the HTML file.

By addressing these issues, I successfully created and published the web page using GitHub Pages.
